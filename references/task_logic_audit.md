# Task Logic Audit: Iowa Gambling Task

## 1. Paradigm Intent

- Task: `iowa_gambling`
- Primary construct: affective decision-making under uncertain reward/punishment contingencies.
- Manipulated factors: participant deck choice (`A/B/C/D`) with deck-specific gain/loss schedules.
- Dependent measures: deck choices over time, advantageous choice rate (`C/D`), reaction time, timeout rate, cumulative net outcome.
- Key citations:
  - `W2035258287` (Li et al., 2009, *Human Brain Mapping*)
  - `W2108913208` (Lin et al., 2007, *Behavioral and Brain Functions*)
  - `W2144547056` (Bolla et al., 2004, *Cerebral Cortex*)
  - `W2155857408` (Fellows, 2004, *Cerebral Cortex*)

## 2. Block/Trial Workflow

### Block Structure

- Total blocks: `2`
- Trials per block: `50`
- Randomization/counterbalancing: block trial labels are generated by `BlockUnit`, but each trial is implemented as free deck choice with all four decks concurrently available.

### Trial State Machine

1. `fixation`
   - Onset trigger: `fixation_onset`
   - Stimuli shown: central fixation cross.
   - Valid keys: `[]`
   - Timeout behavior: auto-advance after jittered fixation duration.
   - Next state: `decision`

2. `decision`
   - Onset trigger: `decision_onset`
   - Stimuli shown together:
     - decision title and current cumulative score
     - four deck cards (`A/B/C/D`) with key mappings
     - strategy hint text
   - Valid keys: `[deck_a_key, deck_b_key, deck_c_key, deck_d_key]` (`d/f/j/k` by default)
   - Timeout behavior: emit `choice_timeout` and continue without drawing a card.
   - Response triggers:
     - `choice_deck_a`, `choice_deck_b`, `choice_deck_c`, `choice_deck_d`
   - Next state: `feedback`

3. `feedback`
   - Onset trigger: `feedback_onset`
   - Stimuli shown:
     - `feedback_outcome` if a deck was chosen (gain, loss, net, updated total), or
     - `feedback_timeout` if no response.
   - Valid keys: `[]`
   - Timeout behavior: auto-advance after feedback duration.
   - Next state: `iti`

4. `iti`
   - Onset trigger: `iti_onset`
   - Stimuli shown: fixation cross.
   - Valid keys: `[]`
   - Timeout behavior: auto-advance after jittered ITI.
   - Next state: next trial.

## 3. Condition Semantics

- Condition ID: `free_choice`
  - Participant-facing meaning: participant freely chooses among decks A/B/C/D each trial.
  - Concrete stimulus realization:
    - four simultaneously visible deck cards with stable left-to-right placement
    - per-trial score display before and after choice
  - Outcome rules:
    - each chosen deck emits immediate gain and scheduled loss
    - cumulative total updates as `total += gain - loss`

- Deck-level task factors (not presented as separate trial conditions):
  - `deck_a`, `deck_b`: disadvantageous long-term expected return.
  - `deck_c`, `deck_d`: advantageous long-term expected return.

## 4. Response and Scoring Rules

- Response mapping:
  - `D -> deck A`
  - `F -> deck B`
  - `J -> deck C`
  - `K -> deck D`
- Missing-response policy: timeout trial logs no deck choice, no gain/loss update, and timeout feedback.
- Correctness logic: none (value-learning preference task; no correct/incorrect response).
- Reward/penalty updates:
  - deck draw yields `gain` and `loss`; net outcome is `gain - loss`.
  - cumulative score (`total_money`) is carried across trials and blocks.
- Running metrics:
  - advantageous choice rate (`deck C/D` among responded trials)
  - deck-wise choice counts
  - cumulative net gain/loss
  - timeout rate and mean RT.

## 5. Stimulus Layout Plan

- Decision screen multi-stimulus layout:
  - `decision_title`: top center (`0, 300`)
  - `balance_text`: upper center (`0, 245`)
  - four deck cards as colored rectangles:
    - A: `(-420, 10)`
    - B: `(-140, 10)`
    - C: `(140, 10)`
    - D: `(420, 10)`
  - deck labels and key hints centered on each card (`y=20`)
  - `key_hint`: bottom center (`0, -260`)
- Visual hierarchy:
  - score and title first, then deck options as primary choice objects.
- Readability constraints:
  - all participant-facing text is Chinese with `font: SimHei`.
  - wrap widths and spacing prevent overlap at `1280x720`.

## 7. Inference Log

- Decision: represent IGT as free-choice trials with one runtime condition (`free_choice`) and deck identity logged as trial outcome.
- Why inference was required: literature defines decks as response options rather than block conditions.
- Citation-supported rationale: this preserves canonical IGT behavior where participants select among four decks each trial.

- Decision: use fixed 10-draw repeating loss schedules per deck in config.
- Why inference was required: publications differ in presentation details, but converge on disadvantageous (`A/B`) vs advantageous (`C/D`) long-term contingencies.
- Citation-supported rationale: configured schedules implement the cited immediate-reward vs long-term-outcome contrast and the deck-B loss-frequency issue discussed in `W2108913208`.
